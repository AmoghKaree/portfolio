ark banking system

The ARK Banking System is an intuitive web-based banking application that provides a complete banking experience with account management, secure transactions, and real-time balance tracking. Built with Flask and deployed on Render, the system offers a modern, responsive interface for managing checking and savings accounts with comprehensive transaction history.

TECH STACK

Python 3.13, Flask 3.0.0, HTML5, CSS3, JavaScript, Gunicorn, Render

ROLES

Full-Stack Developer, System Designer, UI/UX Designer

KEY FEATURES

Account Management: User registration with secure password validation, authentication system with Flask sessions, support for multiple account types (checking/savings), and comprehensive account dashboard.

Banking Operations: Deposit and withdrawal functionality, instant transfers between accounts, complete transaction history tracking, and real-time balance updates.

Security: Password requirements (uppercase, numbers, special characters), 30-minute secure session timeout, client-side and server-side form validation, and secure cookie-based session management.

TECHNICAL ARCHITECTURE

The application uses a file-based data persistence system where all data is stored in text files. This design choice was intentional for learning purposes - to practice connecting data across multiple files and improve system design skills. The application is deployed on Render's free tier, which uses ephemeral storage. Data resets after approximately 30 minutes of inactivity, though this can be addressed through paid persistent storage ($1/GB/month) or implementing a GitHub-based makeshift database solution.

CHALLENGES & SOLUTIONS

Data Persistence on Free Tier: Render's free tier uses ephemeral storage, causing data loss on restart. Explored solutions including paid persistent storage and a GitHub-based data storage architecture.

Session Management: Implemented secure Flask sessions with 30-minute timeouts and secure cookie handling to protect user data.

File-Based Database: Designed a robust file reading/writing system to manage user accounts and transactions without traditional SQL databases.

FUTURE ENHANCEMENTS

• Migrate to PostgreSQL for improved data management and scalability
• Implement email verification for new user accounts
• Add two-factor authentication (2FA) for enhanced security
• Create PDF generation for transaction reports and statements
• Develop mobile-responsive improvements and native app version
